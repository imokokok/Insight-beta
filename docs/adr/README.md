# Architecture Decision Records (ADR)

本目录包含 OracleMonitor 项目的架构决策记录。

## 什么是 ADR？

架构决策记录（ADR）是一种轻量级的文档形式，用于记录项目中的重要架构决策。每个 ADR 描述一个决策的上下文、决策本身以及其后果。

## 如何添加新的 ADR

1. 使用下一个可用的编号（例如：0005）
2. 复制以下模板创建新文件
3. 更新此 README 中的列表

### ADR 模板

```markdown
# ADR XXXX: 简短的决策标题

## 状态

提议中 / 已接受 / 已弃用 / 已取代

## 背景

描述决策的上下文和问题。

## 决策

明确的决策内容。

## 理由

解释为什么做出这个决策。

## 后果

### 正面

- 正面影响 1
- 正面影响 2

### 负面

- 负面影响 1
- 负面影响 2

## 替代方案

考虑过的其他选项（可选）。

## 参考

相关链接和文档。
```

## 状态说明

- **提议中**: 正在讨论中，尚未决定
- **已接受**: 已采纳并实施
- **已弃用**: 不再使用，但保留记录
- **已取代**: 被新的 ADR 取代，需要链接到新的 ADR
